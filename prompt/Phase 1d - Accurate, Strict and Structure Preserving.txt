## Structural markers

Preserve document structure using only the following markers:

* `{HEADING}...{/HEADING}` — Surround any underlined typewritten heading with these markers. Use the markers only around the text that is underlined, not the entire line unless the entire line is underlined.
* {LB} — Original line break. Output one {LB} for each line break in the source. Consecutive line breaks should be represented by consecutive {LB} markers (for example, one blank line between two lines of text is represented as {LB}{LB}; two blank lines should be represented as {LB}{LB}{LB}, etc.)
* `{INDENT}` — A line that begins with a visually meaningful indentation relative to the surrounding text.

`{INDENT}` may appear only at the beginning of a line, immediately after `{LB}` (or at the beginning of the transcription if the first line is indented). Do not use `{INDENT}` to represent internal spacing within a line.

Use these markers only where they appear in the source. Do not invent additional structural markers.

### Macron + dot below

These are characters or printed forms that contain both a macron above and a dot below. Treat them as containing both diacritical marks.

Lowercase d with both a macron above and a dot below → d{macron}{dotbelow}

Uppercase D with both a macron above and a dot below → D{macron}{dotbelow}

Alternative uppercase form: an overtyped uppercase D with an additional horizontal stroke through the middle and a dot below → D{macron}{dotbelow}

Pay particular attention to these characters, as OCR models often identify only one of the two diacritical marks.

## Apostrophe rule

Convert apostrophes immediately preceding an Arabic transliterated letter sequence into `{apostrophe-ayn}`.

Example:

`'Awdih` → `{apostrophe-ayn}Awdih`

If the transliterated word takes an English possessive, convert only the transliterated apostrophe.

Example:

`Ji'mih's` → `Ji{apostrophe-ayn}mih's`

Do not convert apostrophes used in ordinary English.

Examples:

don't
didn't
it's
I've
woman's

---

## Uncertain or illegible text

If a character cannot be read confidently, transcribe it as:

`[?]`

Use the same convention for partially illegible words.

Do not guess missing letters.

Never infer missing letters or words from context.

For example, if the page reads

`Salmn`

output

`Salmn`

—not "Salman."
